AutoMoEBox-Sandbox
 
完整可落地项目说明书（中文·GitHub 正式版）
 
本地自进化 MoE AI + 全域沙盘模拟 + 聊天交互中枢 + MOD 生态 + P2P 分布式一体化平台
 
 
 
1. 项目概述
 
1.1 项目定位
 
AutoMoEBox-Sandbox 是一套完全本地运行、无云端、无服务器、无数据库、绿色可移植的全域智能沙盘系统。
核心内置自研轻量级自进化 MoE 模型 AutoMoEBox，从空架构、无预训练权重开始，实现全自动学习、自动训练、自动专家拆分、自动系统优化，并通过 MCP 标准协议 成为整个平台的中央 AI 大脑。
 
平台整合军事、农业、矿业、工业、商业、运输、能源、城市八大可独立运行、可全域联动的模拟体系，支持真实世界地理/卫星/天气/灾害数据；同时内置 AI 自然语言聊天中枢，用户可通过对话查询状态、下达指令、管理 MOD、优化系统、控制全局沙盘。
 
依托 P2P 资源池与分布式计算，实现数据一次获取、全网共享、不重复请求开源平台；全平台文件夹复制即可无缝迁移，是真正可落地、可成长、可商用、可无限扩展的私人数字世界模拟系统。
 
1.2 核心目标
 
- 构建可直接落地运行的轻量级自进化 MoE 智能中枢 AutoMoEBox
- 实现空架构、无预训练、无数据库、纯文件式长期记忆与自学习
- 支持 8 大基础专家 + 数据过载自动拆分、无限水平扩展
- 提供完整 Web 多模态交互 + 对话聊天界面
- 通过 MCP 协议打通 AI 与沙盘全系统、MOD、P2P、外部硬件/软件
- 自动训练策略：根据数据规模自动选择 LoRA/SFT/DPO/蒸馏
- 实现全域沙盘八大系统独立/联动模拟
- 全平台绿色免安装、复制即迁移、普通 PC 可流畅运行
 
1.3 适用场景
 
- 本地私有自进化 AI 助手
- 全域军事/经济/产业沙盘模拟与指挥
- 可成长、可移植的私人数字大脑
- 工业控制、开发工具、自研软件的本地 AI 中枢
- MOD 驱动的开放式模拟平台
- 轻量化端侧 MoE 模型研究与二次开发

4. AutoMoEBox AI 详细设计
 
4.1 模型基础结构
 
- 模型名称：AutoMoEBox
- 初始化：空架构、随机权重、无预训练
- 专家数量：初始 8 个
- 路由机制：Top-1 门控路由
- 运行设备：CPU / GPU 自动切换
- 存储方式：纯文件、无数据库
 
4.2 专家自动拆分机制
 
- 监控：单专家目录数据量
- 阈值：≥50 条记录触发拆分
- 策略：1 父专家 → 2 子专家
- 路由自动更新
- 可无限水平扩展
 
4.3 自学习系统
 
- 学习来源：对话、文档、沙盘规则、MOD、用户指令
- 学习方式：自动读取 → 归档 → 训练
- 存储格式：文本/JSON
- 无需预处理、无需标注、无需建库
 
4.4 自动训练策略
 
- 短文本（<300 字）→ LoRA
- 指令对话（300–2000 字）→ SFT
- 偏好样本（2000–10000 字）→ DPO
- 大量文档（>10000 字）→ 知识蒸馏
 
4.5 长期记忆系统
 
- 所有交互自动保存
- 按时间戳归档
- 启动自动加载
- 可查看、可编辑、可备份
- 直接用于增量微调
 
4.6 MCP 协议接入
 
- 标准接口：/mcp
- 支持：对话、推理、学习、训练、状态、控制
- 可对接沙盘、MOD、外部系统、硬件
 
 
 
5. 全域沙盘系统详细设计
 
5.1 真实世界底层数据
 
- 全球地形、高程、坡度、通行判定
- 道路、铁路、机场、港口、建筑
- 城市、人口、行政边界
- 卫星真实轨道（TLE）
- 实时天气与自然灾害
 
5.2 八大模拟系统（独立+联动）
 
所有系统默认互相影响：
 
- 农业减产 → 粮价上涨 → 民心下降
- 矿产不足 → 工业停产 → 装备断供
- 运输中断 → 物资短缺 → 市场崩溃
- 能源不足 → 全域停电 → 系统瘫痪
- 商业垄断 → 经济控制 → 城市稳定变化
 
5.3 MOD 扩展系统
 
支持 MOD 类型：
 
- 新玩法模块
- 新产业模块
- 真实数据接入
- 真实世界规则
- 商业逻辑与市场模型
- 新地图、新装备、新科技
 
AI 全自动生命周期：
 
1. 加载
2. 安全扫描
3. 兼容性检测
4. 性能负载检测
5. 接入 P2P 资源池
6. 全局同步
7. 运行监控
8. 限流/休眠/禁止
 
 
 
6. AI 聊天对话中枢（交互设计）
 
6.1 UI 位置
 
主界面右侧可折叠悬浮面板：AI 对话中枢
 
6.2 功能
 
- 文字对话输入/输出
- 快捷指令按钮
- 对话历史滚动
- 清屏、重新生成
- 对话自动存档至 /memory
 
6.3 支持指令示例
 
- 当前系统状态？
- 全球粮食产量？
- 自动优化系统
- 显示所有部队
- 关闭高负载MOD
- 同步P2P数据
- 农业怎么玩？
- 记住我优先流畅模式
 
6.4 交互流程
 
1. 用户输入聊天/指令
2. 对话系统上传至 AutoMoEBox
3. AI 意图识别：查询 / 指令 / 学习 / 聊天
4. 执行沙盘控制 / 数据返回 / 模型学习
5. 回复结果并展示
6. 对话存入长期记忆
7. 后台自动增量训练

8. 落地部署步骤（小白可直接执行）
 
8.1 环境准备
 
1. 安装 Python 3.10–3.11
2. 安装依赖：
 
plaintext
  

pip install torch flask
 
 
8.2 启动项目
 
1. 解压完整包
2. 双击 run.bat
3. 系统自动启动：
- AutoMoEBox AI 中枢
- MCP 协议服务
- P2P 资源池
- 沙盘主程序
- AI 聊天对话界面
 
8.3 访问入口
 
- Web 界面：http://127.0.0.1:1984
- MCP 接口：http://127.0.0.1:1984/mcp
- 沙盘客户端：直接启动
 
8.4 自学习使用
 
- 上传文件 → 自动学习 → 自动训练
- 对话交互 → 自动记忆 → 自动微调
- MOD 放入 mods 目录 → AI 自动检测加载
 
8.5 移植与备份
 
- 直接复制整个文件夹
- 新设备只需 Python 环境
- 启动后所有数据、专家、记忆、进度完全保留
 
 
 
9. 技术栈
 
- AI 核心：AutoMoEBox（自进化轻量 MoE）
- 模型框架：PyTorch
- Web 服务：Flask
- 通信协议：MCP 标准接口
- 网络架构：P2P DHT 去中心化
- 存储：纯文件系统（无数据库）
- 多模态：FFmpeg、pdfplumber、Whisper.cpp
- 运行环境：Python 3.10–3.11
- 部署：Windows 一键绿色版
